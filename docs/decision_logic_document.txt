FINAL RECOMMENDATION AI – DECISION LOGIC DOCUMENT
=================================================

SYSTEM NAME:
Final Recommendation AI

OBJECTIVE:
The Final Recommendation AI system is designed to generate
automated hiring recommendations by combining:

1. Unified hiring scores
2. Behavioral analysis
3. Integrity analysis
4. Risk evaluation
5. Confidence scoring

-------------------------------------------------
DECISION CATEGORIES
-------------------------------------------------

1. Selected
2. Hold / Review
3. Rejected

-------------------------------------------------
INPUT SOURCES
-------------------------------------------------

The recommendation engine uses the following inputs:

1. ATS Score
2. Screening Score
3. HR Interview Score
4. Technical Interview Score
5. Machine Test Score
6. Behavioral Score
7. Integrity Score

-------------------------------------------------
UNIFIED HIRING SCORE
-------------------------------------------------

The unified hiring score is generated in Day 51 using
cross-round aggregation logic.

Example:

Final Hiring Score =
(
ATS * Weight +
Screening * Weight +
HR * Weight +
Technical * Weight +
Machine Test * Weight
)

-------------------------------------------------
DECISION LOGIC
-------------------------------------------------

RULE 1 – SELECTED

Candidate is marked as "Selected" if:

- Final Hiring Score >= 85
- Behavioral Score >= 80
- Integrity Score >= 80

Reason:
Candidate shows strong technical, communication,
behavioral, and integrity performance.

-------------------------------------------------

RULE 2 – HOLD / REVIEW

Candidate is marked as "Hold / Review" if:

- Final Hiring Score >= 70
- Integrity Score >= 60

Reason:
Candidate has acceptable performance but requires
manual review or additional validation.

-------------------------------------------------

RULE 3 – REJECTED

Candidate is marked as "Rejected" if:

- Final Hiring Score < 70
OR
- Integrity Score < 60

Reason:
Candidate performance or integrity level does not
meet hiring requirements.

-------------------------------------------------
CONFIDENCE SCORE
-------------------------------------------------

The confidence score represents AI confidence in the
final hiring recommendation.

Formula:

Confidence Score =
(
Final Hiring Score * 0.5 +
Behavioral Score * 0.2 +
Integrity Score * 0.3
)

The final value is normalized between 0–100.

-------------------------------------------------
RISK ANALYSIS
-------------------------------------------------

Risk analysis is performed using behavioral and
integrity indicators.

Risk Categories:

1. Low Risk
2. Behavioral Concern
3. High Integrity Risk

-------------------------------------------------

LOW RISK

Condition:
- Behavioral Score >= 65
- Integrity Score >= 60

Meaning:
Candidate shows stable and acceptable behavior.

-------------------------------------------------

BEHAVIORAL CONCERN

Condition:
- Behavioral Score < 65

Meaning:
Possible distraction, nervousness, or engagement issue.

-------------------------------------------------

HIGH INTEGRITY RISK

Condition:
- Integrity Score < 60

Meaning:
Possible malpractice or integrity-related concern.

-------------------------------------------------
EXPLAINABLE AI FEATURES
-------------------------------------------------

The system provides transparent outputs by exposing:

- Individual scores
- Final hiring score
- Final recommendation
- Confidence score
- Risk factor

This improves explainability and auditability.

-------------------------------------------------
OUTPUT FILES
-------------------------------------------------

1. final_decision_output.json
   → Candidate-wise final decisions

2. decision_report.json
   → Selected / Review / Rejected candidates

3. recommendation_summary.json
   → System execution summary

-------------------------------------------------
SYSTEM STATUS
-------------------------------------------------

Status: Production Ready

The Final Recommendation AI system successfully
combines multi-round interview intelligence into a
single explainable hiring decision framework.
